## 2026-03-29 F1 Code Quality Review
- Review verdict: REJECT.
- High severity: `pulumi/components/integration_tests.py` checks Flux Kustomizations using names that do not match the resources created in `pulumi/components/flux_gitops.py` (`oc-namespaces` vs `namespaces`, `oc-platform-shared` vs `platform-shared`, `oc-platform` vs `oc-demo-platform`).
- High severity: External Secrets E2E checks in `integration_tests.py` use `cr_version="v1beta1"` while the created resources and prerequisite waiter use `external-secrets.io/v1`.
- High severity: `pulumi/policy/__main__.py` has stale/incomplete dev-seed detection and no longer matches the actual OpenBao seed commands generated by `pulumi/values/openbao.py`.
- Medium severity: `enforce-helm-timeouts` only checks `kubernetes:helm.sh/v4:Chart`, so most `helm.v3.Release` resources are outside policy coverage.


## 2026-03-29 — F2 security review issues
- HIGH: `pulumi/templates/openbao_post_start.sh.tpl` + `pulumi/values/openbao.py` allow shell injection because substituted values are inserted into shell syntax without shell-safe escaping.
- MEDIUM: `pulumi/policy/__main__.py` can bypass prod checks by assuming dev when stack detection fails and by matching only a small subset of actual dev-seed commands.
- MEDIUM: `pulumi/components/workflow_plane.py`, `control_plane.py`, and `observability_plane.py` build `command.local.Command` shell strings from interpolated config-derived values.
- LOW: `pulumi/policy/requirements.txt` and `pulumi/pyproject.toml` use range-based dependency constraints rather than exact pinning.

## 2026-03-29 F3: Architecture Review
- **REJECT**: `IntegrationTests` in `pulumi/__main__.py` does not depend on `FluxGitOps`, even though `pulumi/components/integration_tests.py` conditionally creates Flux deployment and Kustomization tests. This can allow tests to run before Flux is ready.
- **REJECT**: CrossGuard coverage is incomplete for the current codebase: `_DEV_SEED_PATTERNS` in `pulumi/policy/__main__.py` only matches a subset of the dev-only OpenBao seed commands in `pulumi/values/openbao.py`, `enforce-helm-timeouts` only checks `kubernetes:helm.sh/v4:Chart`, and unknown stack detection currently fails open to dev mode.
